### 第一步：本地构建（把代码装进集装箱）
**目标**：将你的源代码和环境依赖打包成一个本地镜像。
- 核心命令：
    ```bash
    docker build -t <本地小名> .
    ```
- 逻辑
    - `.` 代表 Dockerfile 所在的当前目录。
    - `t` (Tag) 给镜像起个本地好记的名字（比如 flask-app）。
    - 缓存机制：Docker 会逐层执行指令，没变的层会使用 CACHED，变的层（通常是 COPY . .）会重新运行。

### 第二步：打标签（贴上云端快递单）
**目标**：给本地镜像起一个“长名字”，这个名字包含了它要去的具体目的地。
-  核心命令
    ```bash
    docker tag <本地小名> <区域>-docker.pkg.dev/<项目ID>/<仓库名>/<镜像名>:<版本号>
    ```
- 逻辑
    - 这步不产生新镜像，只是引用。
    - 区域（如 asia-east1）告诉 Docker 往哪个机房发。
    - 项目/仓库 确定在云端的具体文件夹。
    - 版本号（如 v1.0.0）是版本控制的核心，避免覆盖旧版本。

### 第三步：身份认证（获取入门钥匙）
**目标**：让你的本地 Docker 获得向 GCP 写入数据的许可。
- 核心命令
    ```bash
    gcloud auth configure-docker <区域>-docker.pkg.dev
    ```
- 逻辑
    - 这是一个“一劳永逸”的动作，执行后 Docker 会记住该域名的登录凭证。

### 第四步：推送（正式发货）
**目标**：将镜像层上传到 GCP 存储空间。
- 核心命令：
    ```bash
    docker push <刚才那个长名字>
    ```
- 逻辑
    - Docker 会扫描这个镜像的所有层（Layers）。
    - 增量上传：如果某些基础层（比如 Python 系统层）已经在云端存在了，它会显示 Layer already exists，只上传你修改的代码层，速度极快。
